1. Variáveis e Atribuição em P5.js

Uma Variável é definida como um espaço da memória que armazena informações (como números ou textos) que podem ser alteradas ao longo do tempo. Toda variável recebe um rótulo (nome) que serve para acessarmos e modificarmos seu conteúdo. O processo envolve a Declaração (usando a palavra-chave let), que reserva esse espaço na memória, e a Atribuição (símbolo '=') , que insere o valor desejado nessa variável, como mostrado no exemplo let velocidade = 10.

Vídeo Aula
Observação: Fique atento à diferença de propósito:
  • O comando let é usado para declarar (reservar) a variável na memória.
  • O símbolo = é o operador de atribuição, que armazena um valor na variável.
Lembre-se que, ao atribuir um novo valor, o valor anterior é sobrescrito (perdido), o que é essencial para atualizar dados como a posição e velocidade em simulações de física no P5.js.

2. Operações Matemáticas, de Comparação e Estrutura Condicional If

O estudo de operações matemáticas (adição, subtração, multiplicação e divisão) é usado para realizar cálculos. As operações de comparação (maior que, menor que, igual a, diferente de) verificam condições, sempre retornando um resultado lógico (true - verdadeiro ou false - falso). A estrutura condicional if ("se" em inglês) verifica uma condição: se for verdadeira, o código dentro do bloco é executado; se for falsa, o código é ignorado. O uso combinado dessas ferramentas é essencial para criar a lógica e o controle de fluxo na programação, como exemplificado no P5.js com as variáveis de posição.

Vídeo Aula
Observação: É crucial entender que as Operações de Comparação sempre retornam um resultado Booleano (true ou false). Este resultado é a condição que o comando if verifica para decidir se o bloco de comandos deve ser executado ou ignorado, sendo fundamental para o controle de fluxo do código. As operações matemáticas também são usadas para criar as condições a serem verificadas (ex: posicaox + 10 < posicaoy).

3. Movimento Retilíneo Uniforme (MRU)

O Movimento Retilíneo Uniforme (MRU) descreve um objeto que se move em linha reta com velocidade constante. Isso significa que o objeto percorre a mesma distância em intervalos de tempo iguais e não há aceleração. A fórmula fundamental para o MRU é S=S0+v⋅t, onde S é a Posição Final, S0 é a Posição Inicial, v é a Velocidade (constante) e t é o Tempo decorrido.

Vídeo Aula Exercício Complementar
Observação: No P5.js, o princípio do MRU é aplicado no código por meio da atualização contínua da posição do objeto, frame a frame, usando a fórmula simplificada posicao = posicao + velocidade;. A velocidade pode ser um valor positivo para movimento em um sentido ou um valor negativo para movimento no sentido oposto (como da direita para a esquerda). Para criar o efeito visual de movimento, a cada frame, o valor da posição é atualizado, o plano de fundo é redesenhado (apagando o objeto anterior) e a forma (círculo) é redesenhada na nova posição.

4. Movimento Retilíneo Uniforme na Diagonal (MRU na Diagonal)

O Movimento Uniforme na Diagonal é a combinação de dois movimentos independentes: um no eixo horizontal (x) e outro no eixo vertical (y). Isso significa que a velocidade horizontal (Vx) e a velocidade vertical (Vy) são constantes. A cada quadro da animação, o objeto avança um pouco em x e um pouco em y, mantendo a fórmula básica de posição S=S0+V⋅t aplicada em ambas as direções.

Vídeo Aula
Observação: Para que o movimento na diagonal fique perfeitamente alinhado e a bolinha não avance muito mais em uma direção do que na outra, a proporção entre as velocidades (Vx/Vy) deve ser a mesma que a proporção entre a largura (L) e a altura (A) da tela. No P5.js, o movimento é simulado somando-se as velocidades separadamente: posX = posX + velocidadeX; e posY = posY + velocidadeY;. Para o objeto "quicar" nas bordas, a estrutura if é usada para inverter a velocidade (multiplicar por -1) quando a posição do objeto atinge o limite da tela.

5. Movimento Retilíneo Uniformemente Variado (MRUV)

O Movimento Retilíneo Uniformemente Variado (MRUV) é um movimento em linha reta onde a velocidade varia ao longo do tempo, baseada em uma aceleração constante. A aceleração é a taxa de variação da velocidade. Se a aceleração for positiva, a velocidade aumenta. Se a aceleração for negativa, a velocidade diminui (desaceleração). A fórmula da velocidade é dada por V=V0​+a⋅t, onde V é a Velocidade Final, V0​ é a Velocidade Inicial, a é a aceleração e t é o tempo.

Vídeo Aula Exercício Complementar
Observação:No P5.js, o MRUV é aplicado a cada "quadro" ou frame do programa. Para simular a mudança de velocidade e posição causada pela aceleração, duas etapas de atualização são necessárias:
Atualizar a Velocidade: velocidade = velocidade + aceleracao;
Atualizar a Posição: Em seguida, usa-se a nova velocidade para calcular a nova posição: posicao = posicao + velocidade;
O movimento para a esquerda ocorre quando a aceleração é negativa (desaceleração) ou o sinal das operações de atualização é subtraído: velocidade = velocidade - aceleracao;.

6. Queda Livre

Queda Livre é o movimento de um objeto sob a única influência da gravidade. A gravidade é a aceleração, cujo valor aproximado é 9,8 m/s2 (≈10 m/s2). A aceleração da gravidade age sempre na mesma direção: para baixo. A gravidade é considerada positiva e a velocidade inicial (V0) é zero. A fórmula da velocidade é dada por V=V0​+g⋅t, onde V é a Velocidade Final, V0​ é a Velocidade Inicial, g é a gravidade e t é o tempo.

Vídeo Aula Exercício Complementar
Observação: Na aplicação em P5.js, a cada frame (quadro), o programa simula a Queda Livre calculando a nova velocidade do objeto usando a gravidade. Em seguida, ele usa essa nova velocidade para calcular a nova posição. Como a gravidade afeta a coordenada vertical, as operações de atualização de movimento são:
Atualiza a Velocidade: velocidade = velocidade + gravidade;
Atualiza a Posição: posicaoY = posicaoY + velocidade;

7. Lançamento Vertical

No Lançamento Vertical, quando um objeto é lançado para cima, sua velocidade inicial é positiva, mas a gravidade age como uma aceleração negativa, diminuindo a velocidade a cada instante. Quando a velocidade do objeto chega a zero (V=0), ele para no ar. A gravidade inverte o jogo, desacelerando o corpo na subida e, em seguida, acelerando-o na descida. A fórmula da velocidade é V=V0​+g⋅t

Vídeo Aula Exercício Complementar
Observação: No P5.js, o efeito de subida e descida é criado através da atualização sequencial de velocidade e posição a cada frame:
Atualiza a Velocidade (Subida): A gravidade é subtraída da velocidade (aceleração negativa): velocidade = velocidade - gravidade;
Atualiza a Posição: O valor da velocidade é subtraído da posição Y para que o objeto suba (pois no P5.js o eixo Y cresce para baixo): posicaoY = posicaoY - velocidade;
Quando a velocidade se torna negativa, o objeto começa a descer e a gravidade continua a ser subtraída da velocidade, mas como a posicaoY também está sendo subtraída por um valor de velocidade negativo, a posição Y aumenta, simulando a queda.

8. Lançamento Horizontal

Summary: O Lançamento Horizontal é a combinação de dois movimentos independentes:
Eixo X (Horizontal): O movimento é Uniforme porque a velocidade (Vx) é constante (não há aceleração horizontal agindo). A cada segundo, o objeto avança a mesma distância para o lado, seguindo a fórmula S=S0​+V⋅t.
Eixo Y (Vertical): O movimento é de Queda Livre porque a velocidade (Vy) muda, já que a gravidade acelera o objeto para baixo. A fórmula da velocidade é V=V0​+g⋅t.

Vídeo Aula Exercício Complementar
Observação: No P5.js, o Lançamento Horizontal é simulado atualizando-se as posições X e Y separadamente a cada frame:
Atualização Horizontal (MRU): A posição X é atualizada com a velocidade X, que não muda: posicaoX = posicaoX + velocidadeX;
Atualização Vertical (Queda Livre): A velocidade Y é sempre afetada pela gravidade, sendo atualizada antes de calcular a nova posição Y:
- velocidadeY = velocidadeY + gravidade;
- posicaoY = posicaoY + velocidadeY;

9. Lançamento Oblíquo

O Lançamento Oblíquo é a combinação de dois movimentos que já conhecemos: o Movimento Uniforme na horizontal e o Lançamento Vertical. A diferença é que o objeto é lançado com uma velocidade inicial (V0​) que tem uma inclinação, ou seja, um ângulo (θ).
Velocidade Horizontal (Vx): É constante durante todo o movimento. É calculada pela decomposição: Vx=V0​⋅cos(θ).
Velocidade Vertical (Vy): Muda por causa da gravidade. É calculada pela decomposição: Vy=V0​⋅sin(θ).

Vídeo Aula Exercício Complementar
Observação: No P5.js, o lançamento oblíquo exige o uso de trigonometria:
Cálculo Inicial (Setup): As velocidades iniciais Vx e Vy são calculadas na function setup usando as funções cos(), sin() e radians().
Atualização (Draw): Na function draw, o movimento horizontal é MRU e o vertical é Lançamento Vertical (ou Queda Livre):
- posicaoX = posicaoX + velocidadeX; (MRU Horizontal);
- velocidadeY = velocidadeY - gravidade; (MRUV Vertical);
- posicaoY = posicaoY - velocidadeY; (Atualização da Posição Vertical).

10. Estrutura Condicional If-Else

A estrutura if-else (Se-Senão) é usada quando sua lógica de programação exige uma escolha binária (uma coisa ou outra).

Vídeo Aula
Observação: É possível aninhar estruturas if-else para criar lógica com múltiplas condições (ex: if (sinal_é_verde) { Avança } else { if (sinal_é_amarelo) { Atenção } else { Para } }). No P5.js, o if-else é muito útil para mudar o estado visual da tela, como alterar a cor de fundo baseado em uma condição. Por exemplo, se a posição X for menor que a metade da largura, o fundo fica azul claro, senão, fica azul escuro.
